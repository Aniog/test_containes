# ARTITECT MACHINERY Design Guide

## Brand Direction
- Tone: elegant, precise, industrial, trustworthy, user friendly
- Audience: factory owners, fabrication teams, distributors, and buyers comparing folding machinery
- Goal: make technical machinery feel premium, approachable, and easy to understand

## Visual Style
- Use a bright, refined industrial look with soft stone backgrounds and deep slate text
- Accent color should feel premium and engineered, using amber/gold sparingly for emphasis
- Prefer clean sections with strong spacing, subtle borders, and restrained shadows
- Example Tailwind palette: `bg-stone-950`, `bg-stone-100`, `bg-white`, `text-slate-900`, `text-slate-600`, `border-stone-300`, `text-amber-500`, `bg-amber-500`

## Typography
- Primary font: Inter
- Headings: bold, compact, confident
- Body: medium line height, easy to scan
- Example classes: `text-4xl md:text-6xl font-semibold tracking-tight`, `text-base md:text-lg leading-7 text-slate-600`

## Layout
- Use generous horizontal spacing with centered content containers
- Desktop layouts should feel structured and premium, with multiple columns where appropriate
- Mobile layouts should stack cleanly with comfortable spacing
- Example classes: `mx-auto max-w-7xl px-6 lg:px-8`, `grid gap-6 lg:grid-cols-3`

## Surfaces
- Cards should use white or very light stone backgrounds with readable dark text
- Include rounded corners, subtle borders, and soft shadows
- Example classes: `rounded-3xl border border-stone-200 bg-white shadow-sm`

## Components
- Buttons should feel strong and clear, with one dark primary button and one light secondary button
- Product/spec blocks should be clear and scannable
- Use icon-supported highlights for capabilities and trust indicators

## Do
- Keep all text highly readable against its background
- Use premium spacing and consistent alignment
- Present technical information in digestible groups
- Emphasize reliability, precision, and operator ease

## Don't
- Do not use neon colors or playful styling
- Do not crowd the page with dense technical tables
- Do not use weak-contrast text on stone or white surfaces
- Do not make desktop sections look like narrow mobile stacks
